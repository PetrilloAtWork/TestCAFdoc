Detailed Description

Interface with ICARUS channel mapping SQLite database.

This interface fills "standard" data structures from a SQLite database. These data structures are then managed by a ICARUS service provider (icarusDB::IICARUSChannelMap).

Reads a full table from the database and process data with func.

Parameters
tablename of the table to read
funccallback function to be called to process the fetched data
datapointer to a memory data region
Returns
an error code, SQLITE_OK on success

The full content of the table named table is read from the configured database. A connection to the database is opened, used and finally closed.

The callback function func is executed with the newly loaded data. The pointer data is passed to that function, which can reinterpret it and modify it in the usual untyped C-style data transfer.

◆ SelectPeriod()

bool icarusDB::ChannelMapSQLite::SelectPeriod ( RunPeriod period)
overridevirtual

Prepares the object for queries pertaining the specified period.

Parameters
periodthe period to be prepared for
Returns
whether values cached from the previous period are invalidated

Periods are defined in icarusDB::RunPeriods class.

If the return value is true, caller should follow by rebuilding all the maps they need to use. If the return value is false, all maps built after the previous call to SelectPeriod() are still current for period: building new ones is not necessary, but it's not harmful either (at most, wasteful).
